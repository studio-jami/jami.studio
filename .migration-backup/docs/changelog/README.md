# Changelog Fragments

Production-meaningful changes to the marketing site get a dated Markdown fragment here, named
`YYYY-MM-DD-short-slug.md`. Fragments are the human-readable record of what shipped and why;
git history is the full archive. Keep them short: what changed, why, and any operator follow-up.

Do not put secrets, tokens, or analytics keys in fragments (the repo-wide rule). Reference
durable docs under `docs/` rather than restating them.
